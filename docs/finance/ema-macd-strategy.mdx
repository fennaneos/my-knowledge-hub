This note explains a simple trend-following strategy using **Exponential Moving Averages (EMAs)** and **MACD**.

## Exponential Moving Average (EMA)

For a close series \(C_t\) and period \(n\), define the smoothing factor
$$
\alpha = \frac{2}{n+1}.
$$
The EMA is updated recursively as
$$
\mathrm{EMA}_t = \alpha\, C_t + (1-\alpha)\, \mathrm{EMA}_{t-1}.
$$

Common pairs: fast = 12, slow = 26.

## MACD

The MACD line is the difference between fast and slow EMAs:
$$
\mathrm{MACD}_t = \mathrm{EMA}^{\text{fast}}_t - \mathrm{EMA}^{\text{slow}}_t.
$$

The **Signal** line is an EMA of the MACD, typically period 9:
$$
\mathrm{Signal}_t = \mathrm{EMA}_{9}(\mathrm{MACD}_t).
$$

A common histogram is
$$
\mathrm{Hist}_t = \mathrm{MACD}_t - \mathrm{Signal}_t.
$$

## Basic Rules

- **Bullish**: fast EMA crosses **above** slow EMA (or MACD crosses above Signal) → bias long.  
- **Bearish**: fast EMA crosses **below** slow EMA (or MACD crosses below Signal) → bias short.  
- Optional filter: require price above/below a higher-timeframe EMA.

## Pseudocode

```text
inputs: fast=12, slow=26, signal=9

for each bar t:
  emaFast[t] = EMA(close, fast)
  emaSlow[t] = EMA(close, slow)
  macd[t]    = emaFast[t] - emaSlow[t]
  sig[t]     = EMA(macd, signal)

  crossUp    = (emaFast[t-1] <= emaSlow[t-1]) and (emaFast[t] > emaSlow[t])
  crossDown  = (emaFast[t-1] >= emaSlow[t-1]) and (emaFast[t] < emaSlow[t])

  if crossUp:   go long (or reduce shorts)
  if crossDown: go short (or reduce longs)
